II.10.1 HTML.Añadir CSS

El CSS es un lenguaje de diseño gráfico que permite aplicar estilos al HTML. Sus siglas significan "hojas de estilo en cascada" (traducción del inglés Cascading Style Sheets). Este lenguaje es muy utilizado para establecer la apariencia visual de una página web.

Ventajas del CSS

Gracias al CSS es posible definir el color de los elementos HTML, ya sea el color de fondo, de los bordes, de las sombras o del texto. También es posible definir el espaciado y el alineamiento de los elementos, configurar las fuentes y todo lo que tenga que ver con la apariencia y el diseño de una página web.

Formas de añadir CSS a HTML

MétodoMedioDescripción
CSS en líneaMediante el atributo styleInserta código CSS directamente sobre un elemento HTML específico.
CSS internoMediante el elemento <style>Los estilos se aplican cuando el documento se carga.
CSS externoMediante el elemento <link>Enlaza un archivo CSS externo.

Atributos y elementos utilizados

Elemento/AtributoDescripción
stylePara insertar código CSS en línea, sobre un elemento HTML.
<style>Para insertar código CSS en el documento actual.
<link>Para incrustar un recurso externo con código CSS.

Selectores CSS

Los selectores son fundamentales para aplicar estilos a los elementos HTML. Existen tres tipos principales:

Tipo de selectorUso
Atributo idSelector único. Ejemplo: #foo { color: red }
Atributo classSelector múltiple. Ejemplo: .centrado { text-align: center }
Selectores de tipoAplican a todos los elementos del mismo tipo. Ejemplo: p { color: blue }

II.10.2 HTML.Añadir JS

JavaScript (JS) es un lenguaje de programación interpretado que permite añadir interactividad a una página web. Es muy usado en el lado del cliente, aunque también puede usarse en el lado del servidor.

Hay dos maneras de añadir código JavaScript a un documento HTML o página web:

Método Descripción
Elemento <script> Permite insertar código directamente en el documento actual o enlazar un recurso externo.
Atributos de eventos Se usan para ejecutar acciones concretas mediante código JS cuando ocurren ciertos eventos.

Añadir código JS: elemento <script>

El elemento <script> permite insertar código JavaScript directamente en el documento HTML entre las etiquetas <script> y </script>.

Ejemplo:


            <script>
                alert('Hola')
            </script>
Adicionar JS desde archivo externo

También se puede enlazar un archivo JavaScript externo usando el atributo src del elemento <script>.

Ejemplo:

<script src='archivo.js'></script>

El atributo src especifica la ruta del archivo JavaScript externo. Es importante que el archivo tenga la extensión .js.

El atributo defer permite que el script se ejecute después de que el documento HTML se haya cargado completamente, lo que mejora la carga del documento.

Ejemplo:

<script src='archivo.js' defer></script>

El atributo async permite que el script se cargue de forma asíncrona, lo que significa que no detiene la carga del documento HTML mientras se descarga el script.

Ejemplo:

<script src='archivo.js' async></script>

Ejemplo completo:


            //Archivo: reloj.js 
            // Muestra la hora actual en un elemento con id="hora"
            function mostrarHora() {
                const ahora = new Date();
                const hora = ahora.toLocaleTimeString();
                document.getElementById('hora').textContent = hora;
            }
            setInterval(mostrarHora, 1000);
            window.onload = mostrarHora;
        

            <p>Hora actual: <span id="hora"></span></p>
            <script src="./RECURSO/reloj.js"></script>

Hora actual:

Atributos del elemento <script>

Los atributos permiten configurar funcionalidades avanzadas para mejorar la carga del documento HTML. Algunos de los más importantes son:

Atributo Descripción
src Especifica la ruta del archivo JavaScript externo (.js).
type Especifica el tipo de contenido, por defecto es text/javascript.
nomodule Evita que el código se ejecute en navegadores que soportan módulos.
async Carga el script de forma asincrónica sin detener la carga del documento.
defer Hace que el script se cargue después de que termine de cargarse todo el documento.
crossorigin Controla las credenciales CORS al cargar recursos desde otro dominio.
integrity Verifica la integridad del recurso con un hash seguro.
referrerpolicy Especifica cómo se envía la información del referente al cargar recursos.

Añadir código JS con atributos de eventos

Los elementos HTML pueden tener atributos de eventos que ejecutan código JavaScript cuando ocurre una acción, como hacer clic, mover el ratón o cargar la página.

Ver ejemplo de evento onclick en HTML

Ejemplo:
<p onclick='alert("Mensaje de alerta.")'>Pulsa este párrafo.</p>

Resultado en el navegador:

Pulsa este párrafo.

El elemento <noscript> y su utilidad

El elemento <noscript> se usa para mostrar contenido alternativo si el navegador no tiene soporte para JavaScript o está deshabilitado. Tiene etiquetas de apertura y cierre obligatorias.

Ejemplo:


            <script>
                alert('Se muestra si tiene JS habilitado.')
            </script>
            <noscript>Se muestra si tiene JS deshabilitado.
            </noscript>

II.10.3 HTML.Elementos <link>

El elemento link permite proveer información relacionada entre el documento HTML actual y un recurso externo enlazado, como pueden ser hojas de estilo en cascada (CSS), fuentes o tipografías, y los iconos de página (favicon).

Estructura básica del elemento link

La etiqueta del elemento es <link>. Este elemento no tiene etiqueta de cierre, por lo que solo se usa la etiqueta de apertura. Debe estar anidado dentro de las etiquetas del elemento <head>.

Ejemplo:

<head>
  <link rel='stylesheet' href='estilos.css'>
  <link rel='icon' type='image/x-icon' href='favicon.ico'>
</head>

Atributo href

El atributo href define la URL del recurso enlazado. Puede ser absoluta o relativa. Es obligatorio usar este atributo.

Atributo rel

El atributo rel indica la relación entre el documento actual y el recurso enlazado. Algunos valores comunes incluyen:

  • stylesheet: para hojas de estilo CSS.
  • icon: para favicones.
  • preload: para recursos que se precargan.
  • canonical: para evitar contenido duplicado.

Atributo crossorigin

Este atributo controla si se envían credenciales CORS al hacer una solicitud de origen cruzado. Sus valores posibles son:

  • anonymous: sin enviar credenciales.
  • use-credentials: enviando credenciales.

Atributo media

Especifica para qué tipo de medio está diseñado el recurso. Valores comunes:

  • print: diseño para impresión.
  • screen: diseño para pantallas.
  • all: para todos los tipos de medio (valor por defecto).

Atributo integrity

Verifica la integridad del recurso enlazado con un hash codificado en Base64. Solo se usa cuando rel tiene valores como stylesheet, preload o modulepreload.

Ejemplo:

<link rel='stylesheet' href='style.css' integrity='sha256-yLGHRkK85JGUMJsevPaKCQ4gYknL2SIGgERVBSiP/Ks='>

Atributo hreflang

Especifica el idioma del recurso enlazado. Se recomienda usarlo junto con el atributo href.

Atributo type

Indica el tipo MIME del recurso enlazado. Ejemplos:

  • text/css: para archivos CSS.
  • image/x-icon: para iconos.

Atributo referrerpolicy

Controla cómo se envía la referencia al servidor. Valores principales:

  • no-referrer: no se envía ninguna referencia.
  • no-referrer-when-downgrade: valor predeterminado.
  • origin: solo se envía el origen.
  • unsafe-url: se envía toda la URL (no recomendado por seguridad).

Atributo sizes

Define los tamaños disponibles de un icono. Se usa cuando rel tiene valores como icon o apple-touch-icon.

Ejemplo:

<link rel='icon' href='icono.png' sizes='16x16 32x32 64x64'>

Atributo imagesrcset

Similar al atributo srcset de imágenes, permite definir múltiples versiones del icono según tamaño de dispositivo.

Ejemplo:

<link rel='icon' href='icono.png' imagesrcset='icono-400.png 400w, icono-500.png 500w'>

Atributo imagesizes

Funciona como el atributo sizes de imágenes. Define qué imagen usar dependiendo del tamaño del dispositivo.

Ejemplo:

<link rel='icon' href='icono.png' imagesrcset='icono-400.png 400w, icono-500.png 500w' imagesizes='(max-width: 400px) 400px, 500px'>

Atributo as

Especifica el propósito semántico del recurso enlazado. Se usa cuando rel tiene valores como preload o prefetch. Algunos valores posibles:

  • font: para fuentes web.
  • image: para imágenes.
  • script: para archivos JavaScript.
  • style: para hojas de estilo.

II.10.4 HTML.Elementos <meta>

El elemento <meta> permite especificar información adicional importante sobre el documento HTML o página web. La información de este elemento es invisible para los usuarios, pues está pensada para los navegadores web u otras aplicaciones que puedan necesitar información complementaria.

La etiqueta de este elemento es <meta>; sin etiqueta de cierre. Por tanto, se trata de un elemento vacío. El contenido de este elemento se introduce por medio de sus atributos. También es conocido con el nombre de etiqueta meta o metaetiqueta; el nombre es una traducción de Meta-Tag.

Se puede introducir tantos elementos <meta> como sean necesarios. Teniendo en cuenta que no son vistos por el usuario, su uso no afectará a la apariencia del documento HTML. Para introducirlos, es obligatorio que estén anidados dentro de las etiquetas del elemento <head>.

Atributos

Atributo Valor Descripción
charset Codificación de caracteres Especifica la codificación de caracteres del documento HTML.
name application-name, author, description, generator, keywords, viewport, robots Especifica el nombre de la información adicional.
http-equiv content-language, content-type, content-style-type, content-security-policy, refresh, allow, content-encoding, content-length, date, expires, cache-control, last-modified, location, set-cookie, X-UA-Compatible, default-style Especifica una cabecera HTTP.
content Cadena de texto Especifica el valor asociado con el atributo name o con http-equiv.

Ejemplos comunes del elemento <meta>

Tipo Ejemplo Uso
Codificación de caracteres (charset) <meta charset='UTF-8'> Especifica la codificación del documento.
Directiva pragma <meta http-equiv='Content-Type' content='text/html; charset=UTF-8'> Otra forma de definir la codificación.
Descripción de la página <meta name='description' content='Esta página web explica los algoritmos iterativos.'> Usado en SEO para describir el sitio.
Palabras clave <meta name='keywords' content='algoritmos, programación, Python'> Define palabras clave relevantes para el contenido.
Autoría <meta name='author' content='Isaac Newton'> Especifica el autor del documento.
Control de indexado <meta name='robots' content='noindex'> Evita que buscadores indexen la página.
Recarga automática <meta http-equiv='refresh' content='30'> Vuelve a cargar la página cada 30 segundos.
Política de seguridad <meta http-equiv='content-security-policy' content='default-src https:'> Controla qué recursos carga la página.
Hojas de estilo por defecto <meta http-equiv='default-style' content='/style.css'> Define una hoja de estilo predeterminada.
Fecha de vencimiento <meta http-equiv='expires' content='2020-01-01'> Indica cuándo expira la página.

Consideraciones importantes

La especificación de la codificación debe estar dentro de los primeros 1024 bytes del documento HTML. Por lo tanto, lo mejor es colocarla inmediatamente después de la etiqueta de apertura del elemento <head>. En un documento HTML debe haber como mínimo una codificación especificada, pero nunca de las dos maneras a la vez.
